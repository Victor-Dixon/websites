## Global Rules

### Typography
- `casing`: lowercase_preferred
- `line_breaks`: intentional
- `paragraph_lines_max`: 3
- `punctuation`:
  - `allow_em_dash`: false (use ellipses instead)
  - `ellipses_rate`: low
  - `commas`: minimal

### Content
- `emoji`: frequency: never
- `tone_constraints`:
  - no hype
  - no marketing language
  - no jokes unless they serve clarity
  - no corporate phrasing

## Tonal Character

### Adjectives
- measured
- grounded
- reflective
- precise
- quietly confident

### Emotional Range
- calm under pressure
- honest about failure
- respectful of effort
- aware of time passing

## Narrative Position
- `stance`: inside the system
- `perspective`: first-person observer
- `time_awareness`: explicit
- `authority_source`: consistency, not claims

## Structural Rules

### Required Sections
1. `world_state` - current system condition
2. `what_changed` - what was modified
3. `what_held` - what remained stable
4. `what_failed` - what did not work
5. `artifacts_created` - outputs generated
6. `open_loops` - unresolved elements

### Optional Sections
- `why_this_matters` - significance explanation
- `future_readers_note` - guidance for later builders

## Section Styles

### world_state
```
tone: status_report
example: "world state: unstable but improving"
```

### what_changed
```
tone: factual
verbs: ["changed", "removed", "introduced", "rewired"]
```

### what_held
```
tone: affirming
purpose: identify stability
```

### what_failed
```
tone: neutral
rules:
- no blame
- no dramatization
```

### artifacts_created
```
tone: inventory
format: bulleted
```

### open_loops
```
tone: unresolved
verbs: ["remains", "persists", "needs pressure"]
```
