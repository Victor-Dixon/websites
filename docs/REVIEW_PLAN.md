## Review Workflow

### Step 1: Tag Repository
- Create `tags/[REPO_NAME].tags` file
- Add location, status, and category tags
- Update MASTER_CATALOG.md

### Step 2: Professional Review
- Use `reviews/.review_template.md`
- Complete automated + manual review
- Document findings, IP logic, patterns
- Save as `reviews/[REPO_NAME]_review.md`

### Step 3: Grade Card
- Use `grade-cards/.gradecard_template.md`
- Evaluate all criteria
- Make recommendations
- Save as `grade-cards/[REPO_NAME]_gradecard.md`

### Step 4: Master Task List
- Use `master-task-lists/.task_list_template.md`
- Define "done" for this specific project
- Create roadmap to publication
- Save as `master-task-lists/[REPO_NAME]_tasks.md`

### Step 5: Blog Post
- Use `blogs/.blog_template.md`
- Write publication-ready post
- Save as `blogs/[REPO_NAME]_blog.md`

### Step 6: Decision
- Keep & Publish
- Keep & Archive
- Consolidate
- Delete
