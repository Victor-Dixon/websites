Onboarding

HOW TO
START.

Dream.OS is a self-healing execution system for builders turning chaos into proof.
Three steps. No excuses.

01
Pick a Mission

You don't start with a goal. You start with a single piece of chaos. A broken repo. A stale task list. An idea with no home. Pick one. Name it. That's your mission.

02
Ship Proof

No "I'm working on it." Every mission ends with an artifact: a screenshot, a commit, a deploy, a log, a verified report. If you can't show it, it didn't happen.

03
Post a Closeout

Every task ends with a closeout. Task. Action. Verify. Proof. Status. That's it. That's the culture. Every time.

The Closeout Format
TASK: [what you set out to do]
ACTION: [what you actually did]
VERIFY: [how you confirmed it worked]
PROOF: [screenshot / commit / log / deploy URL]
STATUS: [COMPLETE / BLOCKED / IN REVIEW]
