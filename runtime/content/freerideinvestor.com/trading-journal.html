Trading Journal Workflow

This is not a diary. This is the operator workflow: intake → import → replay → behavior scorecard → next rule.

Core Promise

Import

Bring broker fills, activity exports, or supported trading records into the journal pipeline.

Replay

Match each fill to market context so the system can reconstruct what was happening when you acted.

Score

Generate behavior tags, discipline components, and a next-rule candidate for the next session.

Workflow

Define thesis. What trade idea were you acting on?
Set invalidation. What would prove the trade wrong?
Import fills. Broker records become the evidence source.
Match market context. Each fill receives candle context.
Tag behavior. Rapid repeats, fast flips, chase entries, and weak exits are detected.
Generate scorecard. Patience, flip control, chase control, exit quality, and balance are scored.
Create next rule. The system outputs one behavior rule to test next.

Current Proof

  • 47/47 TSLA option fills matched to market candles.
  • 5,760 one-minute TSLA candles used.
  • 8 behavior tags generated.
  • 62/100 calibrated discipline baseline.
  • Top leak: rapid repeat fills / clustering.

Policy

Exact option P/L remains blocked until broker contract identity is imported.

Valid now: behavior replay, timing analysis, market context, clustering detection, and rule generation.

Blocked for now: exact option contract pairing, exact contract-level P/L, and definitive winner/loser accounting.

Next Rule Candidate

After any debit fill, wait at least one full 1-minute candle close before adding or flipping, unless the original thesis invalidation level is hit.
